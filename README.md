# Dự án Thực hành Git & GitHub - Nhóm 17

Dự án này là bài thực hành Git & GitHub của nhóm 17, tập trung vào quản lý phiên bản và phối hợp làm việc nhóm.

## 1. Thông tin chung

- **Môn học:** Thực hành Git & GitHub.
- **Chủ đề:** Quản lý version code khi làm việc nhóm.
- **Tên Repository:** `group17-github-practice`.
- **Ứng dụng:** Hệ thống quản lý sinh viên đơn giản bằng Python.

## 3. Mục tiêu bài tập

- Biết sử dụng GitHub để làm việc nhóm.
- Thành thạo thao tác tạo nhánh (branch), commit, push và pull.
- Thực hiện quy trình Pull Request (PR) và Merge code.
- Biết cách xử lý xung đột code (conflict) cơ bản.

## 4. Cấu trúc dự án

Dự án bao gồm tối thiểu 3 tệp mã nguồn Python:

- `main.py`: Chứa giao diện menu điều khiển và luồng chính của chương trình.
- `student.py`: Định nghĩa lớp Sinh viên và các logic quản lý dữ liệu.
- `utils.py`: Chứa các hàm tiện ích hỗ trợ (ví dụ: định dạng, kiểm tra nhập liệu).

## 5. Chức năng chính

Chương trình cung cấp các chức năng tối thiểu sau:

1. **Thêm dữ liệu:** Cho phép nhập thông tin sinh viên mới.
2. **Hiển thị dữ liệu:** Xuất danh sách sinh viên ra màn hình.
3. **Tìm kiếm dữ liệu:** Tìm kiếm thông tin sinh viên trong hệ thống.

## 6. Quy trình thực hiện (Workflow)

Các thành viên thực hiện đúng yêu cầu tối thiểu:

- **Tạo nhánh riêng:** Mỗi thành viên có ít nhất 1 branch riêng.
- **Commit:** Mỗi thành viên thực hiện ít nhất 3 commit.
- **Pull Request:** Mỗi thành viên tạo ít nhất 1 Pull Request.
- **Xử lý Conflict:** Tham gia tạo và giải quyết xung đột trên file README.md.
